# 公文格式排版 Skill

## 概述

将用户写好的 Word 文档一键转换为符合党政机关公文格式（GB/T 9704-2012）的版式。
默认使用“红头公文（通知书式）”格式，红头机关名为“深圳市腾讯计算机系统有限公司”，
红头下方第一行自动作为主标题排版。核心引擎为 `scripts/format_engine.py`（基于 python-docx），
全程本地处理、零上传。

## 首次部署（自动执行）

**首次调用此 Skill 时，无需向用户提问，直接执行自动部署：**

1. 运行环境自检：
```bash
python scripts/setup.py check
```

2. 如果检测到 Python 依赖缺失或字体未安装，向用户请求授权：
   > “公文格式工具首次使用需要安装以下内容：
   > - Python 依赖：python-docx、lxml
   > - 公文字体包：仿宋_GB2312、楷体_GB2312、黑体、宋体、Times New Roman 等（约 8 个文件，从公开源下载）
   > 
   > 是否授权安装？”

3. 用户授权后，一键完成全部部署：
```bash
python scripts/setup.py
```

4. 部署完成后提示：“请重启 Word/WPS 后字体才会完全生效。”

**不询问用户安装方式、不询问是否分步安装、不询问字体路径。直接请求授权 → 授权后一键安装全部内容。**

如果用户拒绝授权，告知：“字体未安装时，排版后的文档可能无法正确显示公文字体。可随时说‘安装字体'重新部署。”

## 核心工作流

### 1. 排版主流程

收到用户提供的文档后，按以下步骤执行：

**步骤 1：文件输入与格式转换**

检查文件扩展名：
- `.docx` — 直接处理
- `.wps` / `.doc` — 调用 `scripts/file_converter.py` 检查 LibreOffice 是否可用，
  可用则自动转换为 .docx，不可用则提示用户手动另存为 .docx
- 其他格式 — 提示用户转为 .docx

```bash
# 检查转换能力
python scripts/file_converter.py check

# 转换文件
python scripts/file_converter.py convert 文件.wps
```

**步骤 2：文档预检**

排版前分析文档结构，检查标题层级是否正确标注：

```bash
python scripts/doc_analyzer.py 文档.docx
```

预检会输出：
- 段落总数、表格数、图片数
- 各级标题（标题/标题1-4）的数量分布
- 疑似未标注层级的标题段落

**关键规则**：如果预检报告 `has_heading_styles: false`（未检测到任何标题样式），
向用户发出警告：

> ⚠ 文档未标注标题层级。工具将无法识别标题结构，全部内容会被当作正文处理。
> 请先在 Word/WPS 中将大标题设为“标题”样式，各级小标题设为“标题1/2/3/4”样式。
> 设置好后重新提交文档。

提供三个选项：
1. 回去补设标题样式后重新提交
2. 强行排版（全部当正文处理）
3. 取消

**步骤 3：格式确认（3 页卡片交互，必须执行）**

排版前必须让用户确认格式。采用 **3 页 `ask_followup_question` 卡片**，每页 4 题，共 12 项，
覆盖绝大部分格式需求。用户无需打字，点击选择即可完成全部格式设定。

---

**默认行为**：如果用户未指定格式，直接使用默认格式（带红头公文），机关名为"深圳市腾讯计算机系统有限公司"。
红头下方第一行内容自动作为主标题排版（方正小标宋简体，段前段后各一行，居中）。
标题之下自动识别主送机关（顶格不缩进，段前段后各一行）。
正文统一仿宋_GB2312，行距 28 磅，原有加粗保持不变。
文末自动识别落款署名与日期，右对齐，署名与日期之间空一行。

**步骤 4：执行排版**

用户确认后，调用排版引擎：
```bash
python scripts/format_engine.py 文档.docx 预设.json --mode bundled
```

参数说明：
- 第一个参数：输入 .docx 文件路径
- 第二个参数（可选）：预设 JSON 路径，默认使用 redhead_notice.json（红头公文）
- `--mode bundled|mac|standard`：字体方案，默认 bundled（内置字体+嵌入）
- `--number`：可选，自动生成层次序号

**步骤 5：输出报告**

排版完成后，引擎会输出详细报告，包括：
- 输入/输出文件路径
- 字体方案与嵌入字体列表
- 段落统计（大标题/各级标题/正文/图片/图题/表题/表格数）
- 警告项（需人工核对的提醒）

将报告完整展示给用户。输出文件为 `原文件名-公文版.docx`，位于原文件同目录。

- 覆盖当前预设
- 另存为新模板（需输入名称）

### 3. 批量处理

当用户要求处理多个文件时（如“把这个文件夹里的文档全排版”）：

1. 扫描目录下所有 .docx / .wps / .doc 文件
2. 对 .wps / .doc 尝试自动转换
3. 逐个运行 `doc_analyzer.py` 预检
4. 汇总预检结果，列出每个文件的标题/图表/问题
5. 确认统一预设后批量执行排版
6. 输出汇总报告：每个文件一行（文件名 / 标题数 / 图表数 / 状态 ✅/⚠️）

## 重要注意事项

1. **默认格式**：未指定格式时，使用“红头公文（通知书式）”，机关名“深圳市腾讯计算机系统有限公司”，
   红头下方第一行自动作为主标题排版
2. **不修改原稿**：输出文件命名为 `原文件名-公文版.docx`，原文件保持不变
3. **字体嵌入**：使用 `--mode bundled` 时，输出文档自动嵌入所用字体（embedTrueTypeFonts），
   确保在其他电脑上也能正确显示
4. **元数据清理**：默认清除作者、单位等隐私信息
5. **标题样式要求**：非红头模式下，工具依赖 Word 内置标题样式（标题/标题1-4）识别层级。
   红头模式下，第一行自动作为主标题，其余层级仍需标注样式
6. **WPS 兼容**：WPS 保存的 .docx 文件可直接处理；.wps 格式需 LibreOffice 转换
7. **中文引号**：所有输出文本中的中文引号使用“”（U+201C / U+201D），不使用直引号
